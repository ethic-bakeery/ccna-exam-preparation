Concept — Reading the Routing Table

The routing table is the router's decision database — a list of known networks and how to reach them. Every forwarding decision starts here. Before a router sends a packet anywhere, it consults this table. Understanding every field is the foundation of everything else in IP Connectivity.

ROUTING PROTOCOL CODES — MEMORIZE ALL

Every route entry starts with a one or two letter code identifying how the router learned about that network:

PREFIX AND NETWORK MASK — DEEP DIVE

CIDR notation (like /24) is shorthand for how many bits in the subnet mask are "1"s. The more bits = more specific = smaller network = higher priority in longest prefix match.

  • /8 = 255.0.0.0 → covers 16 million hosts (class A "network")
  • /16 = 255.255.0.0 → covers 65,536 hosts
  • /24 = 255.255.255.0 → covers 254 hosts (most common LAN)
  • /30 = 255.255.255.252 → 2 usable hosts (point-to-point links)
  • /32 = 255.255.255.255 → single host (local interface IP, or host route)
  • /0 = 0.0.0.0 → matches EVERYTHING (default route)

Variably subnetted — when you see "10.0.0.0/8 is variably subnetted, 4 subnets, 2 masks", it means the classful 10.0.0.0/8 network is split into subnets of different sizes. IOS groups them together in the display for readability.

L routes (Local) — when an interface is configured with an IP, IOS automatically creates TWO routing table entries: one C (connected) entry for the entire subnet, and one L (local) entry as a /32 for the router's own IP address. The L route is how the router knows "this IP belongs to ME" — it processes packets destined for its own address rather than forwarding them.

CRITICAL OSPF COST PROBLEM

Default OSPF reference bandwidth is 100 Mbps (10⁸ bps). FastEthernet (100 Mbps) and GigabitEthernet (1000 Mbps) BOTH calculate to cost 1 because the formula rounds down. This means OSPF cannot distinguish between 100 Mbps and 1 Gbps links — it treats them identically!

Fix: auto-cost reference-bandwidth 1000 (for GigE environments) or 10000 (for 10GigE). Must be set on ALL routers in the OSPF domain identically.

GATEWAY OF LAST RESORT

When a router has a default route (0.0.0.0/0) in its routing table, IOS displays: Gateway of last resort is 203.0.113.1 to network 0.0.0.0

This is the "catch-all" — if no more specific route matches the destination, the packet goes here. Without a gateway of last resort, a packet with no matching route is silently dropped and the router sends an ICMP "destination unreachable" message back to the source.

At branch office routers, you almost always configure a default route pointing toward HQ or the ISP. At ISP routers, they NEVER have a default route because they must know the entire internet (BGP full table).

KEY INSIGHT — C vs L Routes

When you configure ip address 192.168.1.1 255.255.255.0 on an interface, IOS creates TWO entries: C for the whole subnet (192.168.1.0/24) — so the router knows other hosts in that subnet are reachable via this interface. L for the router's own IP (192.168.1.1/32) — so the router knows this IP belongs to itself and processes packets destined for it rather than forwarding them.

Trap Watch — 3 Wrong-Answer Traps

❶ "Which route has better AD — OSPF [110/2] or RIP [120/5]?" → OSPF (110 < 120 = lower AD = MORE trusted). The trap is thinking higher = better.

❷ "Is the L route the same as the C route?" → NO. C = the whole subnet. L = the router's own IP as /32. They are different entries for different purposes.

❸ "A router has OSPF AD 110 and the static AD is also [110/0]" → Impossible. Static default is AD 1, NOT 110. Only OSPF uses AD 110. Don't confuse the OSPF AD (110) with cost (2).

Quiz — Section 3.1
Q1. A router's routing table contains the following entries for the same network:
O 10.1.0.0/16 [110/20] via 10.0.0.1

BThe Static route — it has the lowest administrative distance (AD=1)
CThe EIGRP route — EIGRP is the most advanced protocol
DAll three — equal-cost load balancing is used

Static has AD=1, EIGRP has AD=90, OSPF has AD=110. Lower AD = more trusted = wins. The Static route is installed and used. Metric is only compared WITHIN the same protocol. ✓ Correct: C. In the [AD/metric] format, the FIRST number is Administrative Distance (110 = OSPF default AD), and the SECOND is the metric (2 = OSPF cost). Forwarding Decision Logic
Longest prefix match · AD tiebreaker · Metric & ECMP
Concept — How a Router Decides

Every time a packet arrives, the router performs a lookup against its routing table in a strict five-step decision process. The order matters absolutely — you cannot skip a step or apply them in a different order.

THE 5-STEP FORWARDING DECISION

STEP 1 — Drop if no match

If no route matches AND no default route exists → packet dropped silently. ICMP "destination unreachable" sent to source. This is checked last in practice, but conceptually defines the baseline.

STEP 2 — Longest Prefix Match (MOST IMPORTANT)

Find all routes that match the destination IP. Among ALL matching routes, the one with the longest prefix (most specific) wins — regardless of AD, metric, or protocol. /32 beats /24 beats /16 beats /0.

STEP 3 — Administrative Distance (if same prefix length)

If two routes have the same prefix length, compare AD. Lower AD wins. This is how the router chooses between, say, a static route and an OSPF route for the same /24.

STEP 4 — Metric (if same prefix AND same AD)

Same protocol learned the same prefix via multiple paths. Compare metric — lower wins. The winning route is installed in the routing table.

STEP 5 — Equal Cost Load Balancing (ECMP)

Same protocol, same prefix, same AD, same metric → install ALL paths. Traffic is load-balanced. OSPF default = 4 paths. EIGRP default = 4, max = 16 (with variance for unequal-cost).

LONGEST PREFIX MATCH — BINARY EXPLANATION

The routing table "matches" a destination IP if the network bits of the route match the corresponding bits of the destination. The route that matches the most bits (longest prefix) wins. Here's a concrete example:

Destination: 192.168.1.200

RoutePrefix LengthDoes it match?Match Quality
192.168.0.0/16/16 — first 16 bits fixedYes — first 16 bits matchBroad — 65535 other hosts also match
192.168.1.0/24/24 — first 24 bits fixedYes — first 24 bits matchBetter — only 254 other hosts match
192.168.1.200/32/32 — all 32 bits fixedYes — all 32 bits matchWINS — only this ONE host matches
0.0.0.0/0/0 — ZERO bits fixedYes — matches everythingLast resort only — used if no other matches
THE GOLDEN RULE

More specific always wins. A /32 host route for 10.1.1.25 will ALWAYS be used instead of a /24 route for 10.1.1.0/24, even if the /32 is a static route (AD=1) and the /24 is a connected route (AD=0). Prefix length beats AD. Always.

AD AS TIEBREAKER — PRACTICAL EXAMPLE

R1 learns 192.168.5.0/24 via two protocols simultaneously. Both are the same prefix length (/24). Now AD decides:

ProtocolPrefixADMetricWinner?
OSPF192.168.5.0/241105LOSES — installed as "not active"
RIP192.168.5.0/241203LOSES — AD 120 > AD 110

The OSPF and RIP routes are kept in their respective protocol databases but are NOT installed in the IP routing table. To see them: show ip route 192.168.5.0 longer-prefixes

GlobalNet Corp — Dubai Office
Nadia troubleshoots why traffic goes to the wrong server

GlobalNet's network engineer Nadia receives a complaint: PC users trying to reach the management interface at 10.10.10.50 are being redirected to the general server farm instead of the dedicated management server. She checks R1's routing table:

R1# show ip route S 10.10.0.0/16 [1/0] via 172.16.1.1 ← General server farm (AD=1) O 10.10.10.0/24 [110/5] via 192.168.1.2 ← Management subnet (AD=110) S 10.10.10.50/32 [1/0] via 192.168.2.1 ← Management host route

Nadia's analysis using LPM: Packet destined for 10.10.10.50 has THREE matching routes. LPM comparison:

  • 10.10.0.0/16 — matches 16 bits of 10.10.10.50
  • 10.10.10.0/24 — matches 24 bits (more specific!)
  • 10.10.10.50/32 — matches ALL 32 bits (most specific!)

Winner: the /32 host route via 192.168.2.1 — it beats both broader routes regardless of AD. Nadia adds the host route and traffic correctly reaches the management server.

WHAT IF?

What if Nadia had removed the /32 host route? Then the /24 OSPF route would win (24 bits > 16 bits). The /16 static route would lose despite having lower AD (AD 1 vs AD 110) because it has a SHORTER prefix. AD only breaks ties between routes of EQUAL prefix length.

Trap Watch

❶ "Which wins — Static /24 (AD=1) or OSPF /28 (AD=110)?" → OSPF /28 wins! /28 is longer prefix than /24. AD is irrelevant — prefix length is checked FIRST.

❷ "Router has routes to 192.168.1.0/24 via OSPF and via Static. Which wins?" → Static (AD=1 < OSPF 110). Same prefix length, AD decides.

❸ "Two OSPF paths, same cost — what happens?" → ECMP. Both routes installed. Traffic load-balanced. NOT just one path.

Quiz — Section 3.2
Q3. A router has these routes:
192.168.1.0/24 via 10.0.0.1
192.168.1.128/25 via 10.0.0.2
0.0.0.0/0 via 10.0.0.3

BTo 10.0.0.3 — the /24 is more specific

✓ Correct: C. 192.168.1.200 falls in the range 192.168.1.128–192.168.1.255, which is exactly the 192.168.1.128/25 subnet. Q4. A router learns 192.168.10.0/24 via OSPF (AD=110) and via RIP (AD=120). The OSPF metric is 10 and the RIP metric is 3. Which route is installed in the routing table?

IPv4 & IPv6 Static Routing
Default · Network · Host · Floating · IPv6 variants
Concept — Static Routes in Depth

Static routes are manually configured by the network engineer. They don't adapt to topology changes automatically, but they're simple, predictable, and resource-efficient (no routing protocol CPU/memory). Every enterprise network uses at least some static routes — usually for default routes and host-specific paths.

FOUR TYPES — SYNTAX AND PURPOSE

DEFAULT ROUTE (/0)
ip route 0.0.0.0 0.0.0.0 [next-hop]

Matches ANY destination. Used at edge routers. Sets "Gateway of Last Resort". Cisco calls this a "quad-zero route".

NETWORK ROUTE
ip route 192.168.2.0 255.255.255.0 10.0.0.2

Routes to a specific subnet. Most common static route. Replaces a routing protocol for simple topologies.

HOST ROUTE (/32)
ip route 10.1.1.100 255.255.255.255 10.0.0.1

Routes to ONE specific IP. Always wins via LPM. Used for management access, traffic engineering, and service-specific routing.

FLOATING STATIC (BACKUP)
ip route 192.168.2.0 255.255.255.0 10.0.1.2 200

Higher AD than primary route. Stays hidden while primary exists. Automatically activates when primary disappears. The 200 is the AD.

FLOATING STATIC — HOW IT WORKS

The key insight: a route with higher AD only appears in the routing table if NO lower-AD route for the same prefix exists. Think of it as a shadow route waiting in the wings:

  1. Primary route (OSPF AD=110) for 192.168.2.0/24 is in the table
  2. Floating static (AD=200) for same network exists in config but is NOT shown in routing table
  3. OSPF neighbor goes down → OSPF route removed from table
  4. Router checks: any other route for 192.168.2.0/24? YES — floating static!
  5. Floating static (AD=200) is now the best (only) route → installed in table
  6. OSPF comes back → OSPF route (AD=110) wins → floating static hides again
FLOATING STATIC AD VALUE SELECTION

The floating static's AD must be HIGHER than the primary route's AD. But how high? Common choices:

• Floating backup for OSPF (AD=110): set AD to 150 or 200

• Floating backup for EIGRP (AD=90): set AD to 95 or 150

• Floating backup for connected (AD=0): set AD to 1 or higher (a static is AD=1 by default — won't float over connected!)

• Never set AD=255 — that route is permanently disabled ("unreachable")

Concept — OSPF Fundamentals

OSPF (Open Shortest Path First) is a link-state routing protocol — unlike RIP which only knows hop counts, every OSPF router builds a complete map of the entire network topology in its LSDB (Link-State Database). Using Dijkstra's algorithm, each router independently calculates the shortest path to every destination from that same map. The result: all routers converge faster and more accurately than distance-vector protocols.

ROUTER ID SELECTION — 3 RULES IN ORDER

Every OSPF router needs a unique Router ID (RID) to identify itself in the OSPF domain. IOS selects the RID this way (in priority order):

  1. Manual configuration (highest priority): router-id 1.1.1.1 — always use this. Predictable and explicit.
  2. Highest IP on any active loopback interface: Loopbacks are virtual, never go down — reliable for RID purposes. If you have loopbacks at 10.0.0.1 and 172.16.0.1, IOS picks 172.16.0.1.
  3. Highest IP on any active physical interface: Least reliable — the interface could go down, causing OSPF instability.
RID CRITICAL RULES

❶ RID must be UNIQUE in the OSPF domain. Duplicate RIDs cause routes to be overwritten — OSPF breaks silently.

❷ RID change requires process restart: clear ip ospf process — say "yes" at the prompt. Without this, the new RID won't take effect.

❸ RID looks like an IP address but IS NOT one. It's just an identifier. You can set it to 1.1.1.1 even if that IP doesn't exist anywhere in your network.

DR/BDR ELECTION — BROADCAST NETWORKS

On Ethernet (broadcast multi-access), multiple routers may share one segment. Without DR/BDR, each router would form Full adjacency with every other router — creating N×(N-1)/2 adjacencies (with 10 routers: 45 Full adjacencies!). DR/BDR reduces this:

  • DR (Designated Router): Central point for LSA flooding. All other routers form Full adjacency ONLY with DR and BDR. Reduces flooding dramatically.
  • BDR (Backup Designated Router): Hot standby for DR. Forms Full adjacency with all routers. Promotes to DR if DR fails.
  • DROther: All other routers. They form Full adjacency ONLY with DR and BDR. They are 2-Way with each other (NOT Full).

DR ELECTION RULES

  1. Highest OSPF priority wins DR. Default priority = 1. Range: 0–255.
  2. Priority tie → highest Router ID wins DR.
  3. Priority 0 = router can NEVER become DR or BDR. Set deliberately to prevent edge routers from winning.
  4. Election is NON-PREEMPTIVE. If a new router joins with higher priority, it does NOT take over from the existing DR. DR only changes when the current DR fails. This is the critical exam trap.
NON-PREEMPTIVE — EXAM FAVORITE TRAP

Scenario: R1 is the DR (priority 100). You add R2 with priority 200. Q: What happens? A: NOTHING. R1 stays DR. R2 becomes BDR (priority 200 beats existing BDR). Only when R1 fails will R2 become DR.

To FORCE a re-election after changing priority: clear ip ospf process on all routers (or reload them). This reinitializes OSPF and triggers a new election.

First Hop Redundancy Protocols
HSRP · VRRP · GLBP — Gateway redundancy and failover
Concept — The Default Gateway Problem

Every end device (PC, phone, server) has ONE default gateway configured statically. If that gateway router fails — even if a backup router is sitting right there ready to forward traffic — ALL devices lose connectivity. With 500 PCs in an office, that's a catastrophic outage. First Hop Redundancy Protocols exist to solve this problem entirely transparently.

THE FHRP SOLUTION

FHRPs create a virtual router — a phantom router with a virtual IP and virtual MAC. Two or more real routers share this virtual identity. End devices use the virtual IP as their gateway and never need to know which physical router is serving them. When one physical router fails, the other assumes the virtual identity seamlessly.

HSRP — HOT STANDBY ROUTER PROTOCOL (CISCO PROPRIETARY)

HSRP designates one router as Active (forwards all traffic) and one as Standby (monitors Active via Hello packets, ready to take over).

VRRP — VIRTUAL ROUTER REDUNDANCY PROTOCOL (IEEE 802.1118)

VRRP is the open-standard answer to HSRP. Works on any vendor's gear. Key differences from HSRP:

  • Roles: Master (not Active) and Backup (not Standby)
  • Preemption: ENABLED by default — opposite of HSRP
  • Hello timer: 1 second (much faster convergence than HSRP's 3s)
  • Hold time: 3 seconds (vs HSRP's 10s)
  • Multicast: 224.0.0.18
  • Virtual MAC: 0000.5E00.01xx (xx = VRRP group in hex)
  • Owner concept: Router whose real interface IP equals the virtual IP gets priority 255 automatically — it is the "owner"

GLBP — GATEWAY LOAD BALANCING PROTOCOL (CISCO PROPRIETARY)

GLBP is Cisco's advanced FHRP that solves the biggest HSRP/VRRP limitation: wasted standby routers. In HSRP/VRRP, only ONE router forwards — the standby does nothing while the active is up. GLBP makes ALL routers forward simultaneously.

How GLBP achieves this:

  1. All GLBP routers share one virtual IP (same as HSRP)
  2. Each router gets a unique virtual MAC (unlike HSRP where only Active has the virtual MAC)
  3. The AVG (Active Virtual Gateway) — highest priority router — answers ALL ARP requests for the virtual IP
  4. The AVG responds to each ARP with a different virtual MAC (round-robin or weighted)
  5. PC1's ARP gets virtual MAC A → traffic goes through R1. PC2's ARP gets virtual MAC B → traffic goes through R2
  6. Each router forwarding for its virtual MAC is called an AVF (Active Virtual Forwarder)
  7. Up to 4 AVFs per group — up to 4 routers all forwarding simultaneously

Pan-Africa Bank — Accra HQ
Kwame implements HSRP to protect 300 bank tellers from gateway failure

Six months ago, Pan-Africa Bank suffered a 2-hour outage when the core router failed during peak banking hours. All 300 tellers lost access. Network engineer Kwame is now implementing HSRP with object tracking so any future router failure triggers automatic failover.

Kwame's design: R1 = Active (priority 110, preempt on), R2 = Standby (priority 100). Virtual IP = 192.168.10.1 — exactly what tellers already have as their gateway. No PC reconfiguration needed. Object tracking on R1's WAN link: if WAN fails, priority drops from 110 to 90, which is less than R2's 100 — R2 becomes Active automatically.

WHAT IF? — PREEMPTION FORGOTTEN

What if Kwame forgot standby 1 preempt on R1? After R1 fails, R2 correctly becomes Active. When R1 recovers (priority 110), it does NOT reclaim Active — preemption is disabled by default. R2 stays Active with lower priority 100, R1 sits idle. Kwame loses the benefit of having the better router (R1) as the normal active gateway.

Trap Watch — 3 Critical HSRP Traps

❶ "HSRP preemption is enabled by default" → FALSE. HSRP preemption is OFF. VRRP is ON by default. This difference is heavily tested.

❷ "Which FHRP allows load balancing?" → GLBP only. HSRP and VRRP waste the standby router (it does nothing while active is up).

❸ "HSRP group 10 virtual MAC?" → 0000.0C07.AC0A. Convert group number to HEX: 10 decimal = 0A hex. Group 1=AC01, Group 16=AC10, Group 255=ACFF.

Quiz — Section 3.5
Q9. A network uses HSRP. R1 is Active with priority 110 and preemption enabled. R1 fails and R2 (priority 90) becomes Active. R1 recovers. What happens next?
AR2 remains Active — HSRP does not preempt once a router becomes Active
BR1 reclaims Active — preemption is enabled and R1 has higher priority (110 > 90)
CBoth routers become Active simultaneously — dual-active state
DThe router with the lower IP address becomes Active by default
✓ Correct: B. Because preemption is explicitly configured on R1 (standby 1 preempt), when R1 recovers and rejoins the HSRP group, it compares its priority (110) to the current Active router R2 (90). Since R1's priority is higher AND preemption is enabled, R1 takes back the Active role. Without preempt, R1 would stay in Standby even though it has the higher priority — this is the most common HSRP misconception.
Q10. Which FHRP protocol allows ALL routers in the redundancy group to actively forward traffic simultaneously, maximizing router utilization?
AHSRP — using multiple active groups
BVRRP — Master router forwards for all Backup routers
CGLBP — each router has a unique virtual MAC and the AVG load-balances ARP replies
DECMP static routing — not an FHRP protocol
✓ Correct: C. GLBP (Gateway Load Balancing Protocol) is Cisco's FHRP that uniquely allows all routers to forward traffic simultaneously. The AVG (Active Virtual Gateway) assigns different virtual MACs to different client ARP replies — PC1 gets virtual MAC-A pointing to R1, PC2 gets virtual MAC-B pointing to R2. Both routers forward their respective traffic flows. HSRP (A) only has one Active router; using multiple groups requires manual VLAN-to-group mapping (not automatic LB). VRRP (B) also only allows one Master router. ECMP (D) is a routing mechanism, not an FHRP.
