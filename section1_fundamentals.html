Network Components — Role & Function

Every network is built from specialized devices, each solving a specific problem at a specific OSI layer. The CCNA exam tests whether you know what layer each device operates at, what it forwards (MAC frames or IP packets), and what problem it solves.

ROUTER Layer 3
Routes IP packets between different networks using a routing table. Each interface = a separate network/subnet. Breaks up broadcast domains.
⚡ Routers forward based on IP address (Layer 3). Each interface is its own broadcast domain.
L2 SWITCH Layer 2
Forwards Ethernet frames based on MAC addresses. Builds a CAM (Content Addressable Memory) table. Breaks up collision domains but NOT broadcast domains.
🚨 Switches break collision domains. Only routers (or VLANs) break broadcast domains.
L3 SWITCH L2 + L3
Combines switching AND routing in one device. Uses SVIs (Switched Virtual Interfaces) for inter-VLAN routing. Requires ip routing to be enabled.
⚡ ip routing is disabled by default on L3 switches — this is the #1 missed step!
NGFW / IPS L3 – L7
NGFW: Deep packet inspection, app awareness, user identity policies, URL filtering. IPS: Detects AND blocks threats inline.
🚨 IDS = detect only. IPS = detect AND prevent. NGFW includes IPS functionality.
ACCESS POINT Layer 2
Bridges wireless clients to the wired LAN. Two modes: Autonomous (self-managed) or Lightweight (managed by WLC via CAPWAP).
⚡ Lightweight APs use CAPWAP (UDP 5246 control / 5247 data) to communicate with WLC.
CONTROLLERS SDN
DNA Center: Cisco's intent-based SDN controller — automates config, assurance, and security for wired + wireless. WLC: Wireless LAN Controller — manages lightweight APs centrally (CAPWAP).
ENDPOINTS Layer 7
PCs, laptops, smartphones, IoT sensors — any device that originates or terminates traffic. They communicate using application-layer protocols (HTTP, SMTP, DNS, etc.).
SERVERS Layer 7
Provide services: web (HTTP 80 / HTTPS 443), email (SMTP 25), name resolution (DNS 53), IP assignment (DHCP 67/68). Servers listen on well-known ports and respond to client requests.
PoE 802.3af / at / bt
Power over Ethernet — delivers electricity through the Ethernet cable, eliminating separate power supplies for APs, IP phones, cameras.
StandardPowerUse

802.3at (PoE+)30WDual-band APs, video phones

🏥 Real-World Scenario
Hospital Network: The router connects the hospital to the internet and remote clinics. An NGFW at the perimeter inspects every packet for malware before it reaches patient records. L3 switches at distribution route between VLANs (Nursing, Admin, Guests). L2 access switches connect floor devices. PoE-powered APs let nurses use tablets — no electrician needed; power travels through the Ethernet cable. A WLC manages all 60 APs from a single management console.
📋 EXAM-READY SUMMARY — 1.1
  • Routers = Layer 3, forward by IP, break broadcast domains
  • L2 switches = Layer 2, forward by MAC, break collision domains only
  • L3 switches need ip routing enabled to route between VLANs
  • IPS blocks threats inline; IDS only detects — NGFW includes both
  • Lightweight APs require WLC via CAPWAP (UDP 5246/5247)
  • PoE 802.3af = 15.4W | PoE+ 802.3at = 30W | PoE++ 802.3bt = 60-90W
🚨 TRAP: "Which device breaks broadcast domains?" → Routers only (not switches). Switches only break collision domains. Confusing these is the #1 1.1 exam mistake.
🎯 QUIZ — Section 1.1
Q1. A Layer 3 switch has SVIs configured and IPs assigned. Hosts in VLAN 10 still cannot reach hosts in VLAN 20. What is the most likely cause?
✅ B is correct. A Layer 3 switch has routing hardware but routing is disabled by default. Without ip routing, SVIs cannot route between subnets — the switch acts purely as a Layer 2 device. This is the single most commonly missed step in L3 switch labs.
Q2. Which PoE standard delivers up to 30W per port and is suitable for powering a dual-band 802.11ac access point?
✅ B is correct. 802.3at (PoE+) provides 30W, sufficient for modern dual-band APs. 802.3af only provides 15.4W (older single-band APs). 802.3bt provides 60-90W for high-powered devices. 802.3ab defines Gigabit Ethernet over copper — unrelated to PoE.
Q3. What protocol do lightweight access points use to communicate with the Wireless LAN Controller?
✅ B is correct. CAPWAP (RFC 5415) is the current standard replacing the older Cisco-proprietary LWAPP. CAPWAP uses UDP port 5246 for control and 5247 for data tunneling. SNMP is for monitoring. MPLS is for WAN routing.

● TOPIC 1.2

Network Topology Architectures

Network topology defines how devices are physically and logically connected and organized. The CCNA exam expects you to know which topology fits which scenario and understand the traffic flow patterns of each architecture.

1.2.a — Two-Tier (Collapsed Core)

Merges the core and distribution layers into one. Cost-effective for small-to-medium enterprises (<1,000 users). Two layers: Distribution/Core (combined) + Access. Redundant links between the distribution switches provide failover.

⚡ Two-tier = "collapsed core." The separate core layer is eliminated to reduce cost and complexity.
  ANIMATED — Two-Tier Collapsed Core (Access → Distribution/Core → WAN)
🏢 Real-World Use
A medium-sized manufacturing plant with 400 employees and 3 buildings. Two L3 distribution/core switches provide redundancy; access switches on each floor connect workstations, printers, and PoE cameras. No separate core layer needed — cost is kept manageable.
1.2.b — Three-Tier (Core / Distribution / Access)

Classic Cisco hierarchical model for large enterprise campuses. Each layer has a specific, dedicated role:

LayerRoleDeviceKey Rule
CoreHigh-speed backboneHigh-speed L3 switchNO ACLs, NO QoS — speed only
DistributionPolicy + routingL3 switchApply ACLs, QoS, summarization
AccessEnd device connectivityL2 switchPoE, port security, VLAN assignment
⚡ The core layer must NEVER have ACLs or QoS — it must forward as fast as possible. All policy = distribution layer.
  ANIMATED — Three-Tier Hierarchy (Access → Distribution → Core → WAN)
🏛️ Real-World Use
A university campus with 10,000+ students, multiple buildings, and a data center. The core layer (high-speed fibre switches) connects buildings at 40-100Gbps. Distribution switches in each building enforce per-VLAN QoS and ACL policies. Access switches in each classroom provide PoE for laptops and APs.
1.2.c — Spine-Leaf (Data Center)

Modern data center architecture designed for east-west traffic (server-to-server). Every leaf connects to every spine. No leaf-to-leaf or spine-to-spine links. Any server-to-server communication always takes exactly 2 hops — predictable, low latency.

⚡ Spine-leaf = data center. North-south = client→server. East-west = server→server (dominant in modern DCs with microservices). Always 2 hops = consistent latency.
  ANIMATED — Spine-Leaf East-West Traffic (Server-A → Leaf-1 → Spine-1 → Leaf-3 → Server-E)

Connects geographically dispersed sites using service provider infrastructure.

⚡ SD-WAN is the modern WAN architecture Cisco heavily promotes. It overlays software intelligence over any WAN transport (MPLS, internet, 4G/5G).
1.2.e — SOHO (Small Office / Home Office)

Minimal network for home or very small business. Typically a single all-in-one device (combined router/switch/AP) connecting a handful of devices to the internet via ISP. Simple NAT, basic Wi-Fi, DHCP built-in.

🎯 QUIZ — Section 1.2
Q1. A cloud provider needs a data center where any server can communicate with any other server with predictable, consistent latency in always 2 switching hops. Which architecture best satisfies this requirement?
✅ B is correct. Spine-leaf guarantees exactly 2 hops between any two servers (leaf → spine → leaf) providing predictable, low-latency east-west traffic. This is critical for modern data centers running distributed microservices.
Q2. In a three-tier campus design, an engineer proposes applying ACLs and QoS policies at the core layer to simplify the distribution layer. What is wrong with this approach?
✅ A is correct. Cisco's hierarchical design explicitly states: the core layer is for high-speed packet forwarding ONLY. Applying ACLs or QoS at the core introduces processing overhead that slows forwarding and defeats the purpose of a separate core layer. Policy enforcement belongs at the distribution layer.

Physical Interfaces & Cabling

Choosing the right cable type depends on distance requirements, bandwidth needs, cost, and the environment (EMI susceptibility). The CCNA exam tests which cable to choose and why.

SINGLE-MODE FIBER
• Core: 8–10 µm (very thin)
• Single light path — no modal dispersion
• Distance: up to 100 km
• Speed: 10G–100G+
• Light source: Laser (expensive)
• Jacket colour: Yellow
⚙️ USE FOR: long-haul inter-campus, WAN, ISP backbone
MULTIMODE FIBER
• Core: 50–62.5 µm (larger)
• Multiple light paths — modal dispersion
• Distance: up to 550 m (OM4 at 10G)
• Speed: up to 10G
• Light source: LED / VCSEL (cheaper)
• Jacket colour: Orange or Aqua
⚙️ USE FOR: building risers, data center inter-rack, short-haul
COPPER UTP
• Cat5e: 100 m, 1Gbps
• Cat6: 100 m, 10G (up to 55 m at 10G)
• Cat6a: 100 m, 10G reliably
• Susceptible to EMI
• Supports PoE power delivery
• Pinout: T568A or T568B
⚙️ USE FOR: desktop access layer — PC, IP phone, AP within 100 m

1.3.b — Shared Media vs Point-to-Point
FeatureShared Media (Hub era)Point-to-Point (Switch port)
Collision domainOne domain for all portsEach port = separate collision domain
DuplexHalf-duplex, CSMA/CDFull-duplex (no CSMA/CD needed)

📋 EXAM-READY SUMMARY — 1.3
  • SMF = yellow, long distance (100km+), expensive laser — use for WAN/campus links
  • MMF = orange/aqua, short range (550m max at 10G), LED/VCSEL — use inside buildings
  • Copper = 100m limit, PoE-capable, EMI-susceptible — desktop access layer
  • Full-duplex switch ports = no collision domain, no CSMA/CD, dedicated bandwidth
  • Cat6a is preferred for new installations supporting PoE++ and 10G at full 100m
🎯 QUIZ — Section 1.3
Q1. An engineer needs to connect two data centers that are 8 km apart at 10 Gbps. Which cable type is the ONLY viable option?
✅ C is correct. 8 km far exceeds copper (100m max) and MMF (550m at 10G) limits. Only single-mode fiber can carry 10G over distances up to 100 km. SFP direct-attach copper (twinax) is limited to 3–7 meters for rack connections.
Q2. What is the maximum distance for a Cat6a UTP cable running at 10 Gbps?
✅ B is correct. Cat6a (augmented Category 6) supports 10 Gbps at the full 100-meter standard Ethernet distance. Note: regular Cat6 only reliably supports 10G up to 55 meters — the "a" in 6a specifically extends this to 100m. 550m applies to MMF, not copper.

● TOPIC 1.4

Interface & Cable Issues

Interface problems cause degraded performance or complete connectivity failure. The CCNA exam tests your ability to identify the cause of common errors from show interface output.

DUPLEX MISMATCH
One end set to full-duplex, other to half-duplex. Result: late collisions, CRC errors, runts — traffic works but performance is terrible. Usually caused by auto-negotiation failure when one side is hardcoded.
🚨 Late collisions in show interfaces = duplex mismatch. Fix: set both ends to the same duplex OR both to auto.
SPEED MISMATCH
If speeds are hardcoded differently (one at 100Mbps, other at 1000Mbps), the link simply won't come up. The interface shows line protocol down. Unlike duplex mismatch, there's no partial traffic — it's a complete failure.
🚨 Speed mismatch → link is DOWN. Duplex mismatch → link is UP but performing poorly.
CABLE & FRAME ERRORS
Runts: Frames < 64 bytes — from collisions or bad NIC
Giants: Frames > 1518 bytes — misconfigured jumbo frames
CRC errors: Corrupted frames — bad cable or EMI
Input errors: Sum of all input problems
Late collisions: Duplex mismatch hallmark
SW1# show interfaces GigabitEthernet0/1 GigabitEthernet0/1 is up, line protocol is up Hardware is Gigabit Ethernet, address is 0018.ba27.3300 Full-duplex, 1000Mb/s, media type is 10/100/1000BaseTX ↑ Check duplex and speed here — BOTH must match remote end Input queue: 0/75/0/0 (size/max/drops/flushes) 0 input errors, 0 CRC, 0 frame, 0 overrun, 0 ignored ↑ Healthy interface — all zeros 0 output errors, 0 collisions, 0 late collision, 0 deferred ↑ Late collisions > 0 = DUPLEX MISMATCH — investigate immediately 0 lost carrier, 0 no carrier, 0 pause output
⚡ Troubleshooting flow: Link down? → Check cable, speed mismatch. Link up but slow? → Check duplex with show interfaces. Seeing late collisions? → Duplex mismatch — check auto-negotiation settings on both ends.
! Fix 1: Let both ends auto-negotiate (recommended) SW1(config-if)# speed auto SW1(config-if)# duplex auto
! Fix 2: Hardcode matching values on BOTH sides SW1(config-if)# speed 1000 SW1(config-if)# duplex full
! Verify: look for "Full-duplex, 1000Mb/s" and zero late collisions SW1# show interfaces GigabitEthernet0/1 | include duplex|collision Full-duplex, 1000Mb/s 0 output errors, 0 collisions, 0 late collision, 0 deferred
🎯 QUIZ — Section 1.4
Q1. A switch interface shows "up/up" status, but users report very slow performance. The show interfaces output shows increasing "late collisions" and "CRC errors." What is the MOST likely cause?
✅ B is correct. Late collisions on a switch port are the signature symptom of duplex mismatch. The link is up (not a speed mismatch) but the half-duplex side uses CSMA/CD and backs off, while the full-duplex side keeps transmitting — causing late collisions. Speed mismatch would cause the link to be completely down.
Q2. Which error type in show interfaces output indicates received frames smaller than 64 bytes, typically caused by collisions?
✅ A is correct. Runts are frames below the minimum Ethernet frame size of 64 bytes, typically caused by collisions (which truncate frames). Giants are frames over 1518 bytes. CRC errors indicate corrupt frames (bad cable/EMI). Input drops indicate interface congestion.

Both TCP and UDP operate at Layer 4 (Transport) and use port numbers to identify applications. The core difference: TCP guarantees delivery at the cost of overhead; UDP is fast with no guarantees.

🎙️ Real-World Scenario
VoIP Call: Your Cisco IP phone uses UDP/RTP for the voice stream. A lost voice packet 50ms ago is useless — retransmitting it would arrive too late and make the call sound worse, not better. UDP's zero-overhead model is perfect here. Meanwhile, the HTTPS web session you open in parallel uses TCP 443 — every byte of that bank login must arrive exactly, in order.
📋 EXAM-READY SUMMARY — 1.5
  • TCP = connection-oriented, reliable, sequenced, flow-controlled, 20-byte header
  • UDP = connectionless, no ACK, no retransmit, 8-byte header — fastest delivery
  • 3-way handshake: SYN → SYN-ACK → ACK (not SYN→ACK→SYN-ACK)
  • DNS uses BOTH TCP and UDP port 53 (UDP for queries, TCP for zone transfers)
  • VoIP/video = UDP (latency matters more than perfection)
  • File transfer/web/email = TCP (data integrity matters)
🚨 TRAP: The question "Which protocol provides guaranteed delivery?" → TCP, not IP. IP is Layer 3 and is also best-effort. TCP provides reliability on top of IP.
🎯 QUIZ — Section 1.5
Q1. A network engineer is troubleshooting poor VoIP quality. Calls drop syllables. The engineer proposes switching from UDP to TCP for the voice stream. What is wrong with this approach?
✅ B is correct. VoIP uses UDP because retransmitting a 50ms-old voice packet is pointless — by the time it arrives, the conversation has moved on. A retransmitted packet arriving late causes more disruption (jitter) than the original loss. The correct fix for poor VoIP quality is QoS (prioritizing UDP/RTP packets), not switching to TCP.
Q2. A host needs to resolve a hostname to an IP address. Which transport protocol and port does DNS use for this standard query?
✅ B is correct. Standard DNS queries use UDP port 53 because they're small (<512 bytes) and speed matters. TCP port 53 is used for DNS zone transfers (large data transfers between DNS servers) and responses over 512 bytes. DoH (DNS over HTTPS) uses TCP 443 but is a special modern variant.
Q3. What is the correct sequence for a TCP three-way handshake?
✅ B is correct. The TCP three-way handshake always follows: (1) Client sends SYN, (2) Server responds with SYN-ACK (acknowledging client SYN and sending its own SYN), (3) Client sends ACK (acknowledging server's SYN). After step 3, the connection is ESTABLISHED and data transfer can begin.

🎯 QUIZ — Section 1.6
Q1. A /28 subnet is used for 10 hosts. What is the broadcast address of the SECOND /28 subnet carved from 172.16.5.0/24?
✅ B. /28 block size = 16. Subnet 1: .0–.15 (BC=.15). Subnet 2: .16–.31 (BC=.31). The broadcast is always the last address in the block.
Q2. How many usable host addresses does a /27 subnet provide?
✅ B. /27 = 5 host bits. 2^5 = 32 total addresses. Subtract 2 (network + broadcast) = 30 usable hosts.

🎯 QUIZ — Section 1.7
Q1. A PC shows IP address 169.254.20.5/16. What does this indicate?
✅ B. 169.254.0.0/16 is the APIPA (Automatic Private IP Addressing) range. When a device can't reach a DHCP server, it self-assigns an address here so local communication still works. This is always a DHCP failure indicator — check the DHCP server or the cable.
Q2. Which of the following addresses is NOT a valid RFC 1918 private address?
✅ B. 172.32.0.1 is a public IP. The Class B private range is 172.16.0.0 – 172.31.255.255 (/12). 172.32.x.x is one block past the private range boundary. All other options are valid RFC 1918 addresses.

1.9.d — MODIFIED EUI-64 — HOW IT WORKS

EUI-64 auto-generates a 64-bit interface ID from the 48-bit MAC address. Three steps:

MAC address:     AA:BB:CC:DD:EE:FF
Step 1 — Split in half:  AA:BB:CC  |  DD:EE:FF
Step 2 — Insert FFFE:   AA:BB:CC:FF:FE:DD:EE:FF
Step 3 — Flip 7th bit (U/L bit) of first octet:
AA = 10101010 → flip bit 7 → 10101010 = 0xA8
Result:  A8BB:CCFF:FEDD:EEFF
⚡ IOS command: ipv6 address 2001:DB8::/64 eui-64 — router fills in the interface ID automatically from the interface MAC.

Q1. Which IPv6 address type is automatically configured on every IPv6-enabled interface and is required for IPv6 to function?
✅ B. Link-local addresses (FE80::/10) are automatically configured on every IPv6 interface the moment IPv6 is enabled — even if no global address is configured. They are required for neighbour discovery, router advertisements, and routing protocol hello messages.
Q2. An interface MAC is 00:50:79:AA:BB:CC. What is the EUI-64 derived interface ID?
✅ B. MAC = 00:50:79:AA:BB:CC. Split: 00:50:79 | AA:BB:CC. Insert FFFE: 00:50:79:FF:FE:AA:BB:CC. Flip 7th bit of 0x00 (00000000→00000010 = 0x02). Result: 0250:79FF:FEAA:BBCC.

🎯 QUIZ — Section 1.11
Q1. Three APs are deployed in adjacent areas. Which 2.4GHz channel assignment minimises co-channel interference?
✅ B. In 2.4GHz, only channels 1, 6, and 11 have enough separation (25MHz apart) to be non-overlapping. Each channel is 22MHz wide but spaced 5MHz apart — so channels 1 and 2 heavily overlap. Using 1, 6, 11 ensures adjacent APs don't interfere with each other.
Q2. A company requires the most secure wireless encryption available for its enterprise WLAN. Which standard should be implemented?
✅ D. WPA3 is the current strongest standard, introducing SAE (Simultaneous Authentication of Equals) which replaces the Pre-Shared Key handshake and protects against offline dictionary attacks. It also provides forward secrecy. WPA2 with AES is still widely used but WPA3 is the most secure.

● TOPIC 1.13

Switching Concepts

A switch builds a MAC address table (CAM table) by examining the source MAC of every incoming frame, then makes forwarding decisions based on the destination MAC.

MAC LEARNING & AGING

When a frame arrives, switch reads the source MAC → records MAC + ingress port + VLAN in CAM table. Default aging timer: 300 seconds. If no traffic for 5 mins, entry is removed.

FRAME FORWARDING DECISIONS
Known Unicast: Dest MAC in table → forward to specific port
Unknown Unicast: Dest MAC NOT in table → flood all ports except source
Broadcast (FF:FF:…): Flood all ports except source
Multicast: Flood unless IGMP snooping enabled

MAC AddressPortVLANAge

SW1# show mac address-table Mac Address Table Vlan Mac Address Type Ports ---- ----------- ------ ----- (empty)
! Clear the MAC table (triggers flooding) SW1# clear mac address-table dynamic
! Check aging time (default 300s) SW1# show mac address-table aging-time Vlan Aging Time ---- ---------- 1 300
📋 EXAM-READY SUMMARY — 1.13
  • Switch learns source MAC address → port mapping (NOT destination)
  • Default MAC aging time = 300 seconds (5 minutes)
  • Unknown unicast, broadcast, multicast → all get flooded out all ports except source
  • Known unicast → forwarded ONLY to the specific learned port
  • show mac address-table — verify learned MACs and ports
  • clear mac address-table dynamic — flush table, triggers flooding
🚨 TRAP: "What does a switch do with an unknown unicast frame?" → FLOOD (not drop, not ARP). ARP is a Layer 3 function. Switches flood unknown Layer 2 unicast frames.
🎯 QUIZ — Section 1.13
Q1. PC-A sends a frame to PC-B. The switch has no entry for PC-B's MAC address. What does the switch do?
✅ C. This is "unknown unicast flooding." The switch floods out all ports except the source port hoping the destination will respond. When PC-B replies, its source MAC is learned and future frames are forwarded directly. Switches never drop unknown unicast frames — they flood. ARP is a Layer 3/host function, not a switch function.
Q2. What is the default MAC address table aging time on a Cisco Catalyst switch?
✅ C. Default = 300 seconds. Configurable with mac address-table aging-time [seconds]. If a device is silent for 5 minutes, its entry is purged — the next frame to or from that device triggers learning (source) or flooding (destination) again.
