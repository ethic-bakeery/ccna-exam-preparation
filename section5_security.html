Core Security Terminology

Security professionals use precise terminology. Confusing these terms on the exam — or in real life — leads to wrong decisions. Learn each definition, not just the word.

THREAT

Any potential danger — a threat is possible, not yet occurring. A hurricane is a threat. A hacker group is a threat. A disgruntled employee is a threat. Threats have three subtypes: External (hackers, nation-states), Internal (employees, contractors), Environmental (floods, power cuts).

VULNERABILITY

A weakness that could be exploited. Types: Software (CVE-2021-44228 Log4Shell), Configuration (default passwords, open Telnet), Design (WEP protocol weakness), Physical (unlocked server room), Human (no security training).

EXPLOIT

The actual mechanism used to leverage a vulnerability. EternalBlue (CVE-2017-0144) exploited an SMB vulnerability to spread WannaCry ransomware. A zero-day exploit targets a vulnerability with no available patch. A known exploit targets an unpatched but known vulnerability.

RISK

Risk = Threat × Vulnerability × Impact. A server with a known vulnerability (high Vuln), connected to the internet (high Threat), containing credit card data (high Impact) = CRITICAL RISK. Mitigation reduces either likelihood or impact.

ATTACK CATEGORIES — ALL EXAM-RELEVANT

DoS / DDoS

Overwhelm resources. Millions of packets from thousands of compromised hosts. SYN flood (fills TCP half-open table). UDP flood. Amplification attacks (DNS, NTP).

SOCIAL ENGINEERING

Phishing: mass fake email. Spear phishing: targeted. Vishing: voice. Smishing: SMS. Whaling: targets executives. Tailgating: physical access bypass. Pretexting: fabricated scenario.

MALWARE

Virus: self-replicates via files. Worm: self-replicates via network. Trojan: disguised malware. Ransomware: encrypts data. Rootkit: hides deep in OS. Keylogger: records keystrokes.

PASSWORD ATTACKS

Brute force: try every combination. Dictionary: common words. Credential stuffing: use breached lists. Rainbow table: precomputed hashes. Spraying: one password, many accounts.

LAYER 2 ATTACKS

MAC flooding: fills CAM table → hub mode. ARP poisoning: redirect traffic via fake ARP. DHCP starvation: exhaust IP pool. VLAN hopping: bypass segmentation. STP manipulation: become root bridge.

RECONNAISSANCE

Passive: OSINT, whois, DNS enumeration (no contact with target). Active: nmap port scan, banner grabbing, ping sweep (contact with target). ALWAYS precedes attacks.

EXAM TIP

The CIA Triad is tested in every context. A DDoS attack violates Availability. An eavesdropping attack violates Confidentiality. Data modification in transit violates Integrity. Know which attack violates which pillar.

The Human Element — 90% of Breaches Start Here

Technology alone cannot secure a network. The majority of successful breaches involve human error or manipulation. A security program addresses people, processes, AND technology.

USER AWARENESS PROGRAM

  • Security awareness training — minimum annually, ideally quarterly. Covers: phishing recognition, password hygiene, data handling, reporting incidents
  • Phishing simulation campaigns — send fake phishing emails to employees, measure click rate, provide immediate training for those who clicked
  • Acceptable Use Policy (AUP) — signed by all employees: defines permitted/prohibited computer/network use
  • Clean desk policy — no sensitive documents left visible; lock screens when leaving workstation
  • Social engineering awareness — teach employees to verify caller identity; "IT will never ask for your password"

EXAM TIP — Mantrap Definition

A mantrap (also called an airlock) is a physical security control using two doors: the first door must close and lock before the second opens. Only one person at a time. Prevents tailgating. Common in data centers and bank vaults.

MEMORIZE — Environmental Controls

Fire suppression: FM-200 or CO₂ (not water in data centers — destroys equipment). Temperature: 64–80°F (18–27°C). Humidity: 40–60% RH. UPS (Uninterruptible Power Supply) + diesel generator for extended outages.

Concept — Securing Router and Switch CLI Access

A router with no password protection is an open door to your entire network. Default Cisco IOS has NO console password, NO enable password, and allows Telnet (plaintext). Every device must be hardened before deployment.

ENABLE SECRET vs ENABLE PASSWORD

ATTACK — Why "enable password" Is Dangerous

enable password cisco stores the password in running-config as Type 0 (plaintext). Anyone with read access to the config file sees the password. With service password-encryption it becomes Type 7 — but Type 7 uses Vigenere cipher and is trivially reversible by dozens of online tools in seconds. NEVER use enable password.

MITIGATION — Always Use enable secret

enable secret cisco stores as Type 5 (MD5) by default. Better: enable algorithm-type scrypt secret Str0ng! stores as Type 9 (scrypt) — GPU-resistant hashing. If both enable password and enable secret are configured, enable secret always wins.

LOGIN BLOCK-FOR — BRUTE FORCE PROTECTION

The login block-for command watches for failed login attempts. After N failures within a time window, ALL logins are blocked for a period — even from legitimate administrators.

MEMORIZE — login block-for Syntax

login block-for seconds attempts N within seconds
Example: login block-for 60 attempts 3 within 30
= Block for 60 seconds if 3 failures happen within 30 seconds
During block: login quiet-mode access-class 10 allows ONLY ACL 10 hosts to bypass the lockout (for emergency admin access).

Beyond Simple Passwords

Passwords alone are insufficient for modern security. A 12-character password can be cracked in seconds with GPU-based attacks if it follows predictable patterns. The industry has moved toward layered authentication — something you know, have, AND are.

MEMORIZE — Biometric Error Rates

FAR (False Acceptance Rate): incorrectly grants access to unauthorized person
FRR (False Rejection Rate): incorrectly denies access to authorized person
CER/EER (Crossover Error Rate): where FAR = FRR — lower CER = better biometric system
Increasing sensitivity: FAR↓, FRR↑. Decreasing sensitivity: FAR↑, FRR↓.

IPsec Remote Access & Site-to-Site VPN

Concept — Encrypted Tunnels Over Untrusted Networks

A VPN creates an encrypted, authenticated "tunnel" through the internet. Without VPN, data traveling between sites crosses dozens of untrusted routers — any one of them can capture, read, and modify the traffic. IPsec provides confidentiality (encryption), integrity (hashing), and authentication across any network.

AH vs ESP — THE CRITICAL DISTINCTION

PropertyAH (Auth Header) — Protocol 51ESP (Encapsulating Security Payload) — Protocol 50
Confidentiality (encryption)❌ NO✓ YES (AES, 3DES)
Integrity (hashing)✓ YES✓ YES
Authentication✓ YES✓ YES
Anti-replay✓ YES✓ YES
NAT traversal❌ NO (breaks NAT — covers IP header)✓ YES (ESP-NAT-T, UDP 4500)
Real-world useRarely usedAlways used — ESP is the standard

IPSEC MODES — TUNNEL vs TRANSPORT

TRANSPORT MODE

Encrypts payload only. Original IP header preserved.
Packet: [Original IP][ESP][TCP/Data][ESP Trailer]
Use: host-to-host within same network.

TUNNEL MODE (DEFAULT)

Encrypts entire original packet. New IP header added.
Packet: [New IP][ESP][Old IP+TCP/Data][ESP Trailer]
Use: site-to-site VPN, remote access VPN.

IKE PHASES — HOW VPN TUNNELS FORM

IKE PHASE 1 — ISAKMP SA (Bidirectional)

Establishes a secure control channel for Phase 2 negotiation. Negotiates: encryption algorithm, hash algorithm, authentication method (PSK or cert), DH group, SA lifetime.
Main Mode: 6 messages — slower, more secure. Hides identities.
Aggressive Mode: 3 messages — faster, hash sent in clear → offline attack risk.
Result: ONE bidirectional ISAKMP SA. Router shows: QM_IDLE = Phase 1 up.

IKE PHASE 2 — IPSEC SA (Two Unidirectional)

Negotiates actual data encryption parameters. Uses Quick Mode (3 messages). Negotiates: IPsec protocol (ESP), encryption, hash, mode (tunnel), lifetime.
Result: TWO IPsec SAs — one per direction. show crypto ipsec sa shows pkts encaps/decaps.

Concept — How ACLs Work

An ACL is an ordered list of permit/deny statements applied to a router interface. The router checks each arriving or departing packet against the ACL top-to-bottom. The first match wins — processing stops. If no match: the implicit deny any at the end drops the packet.

ATTACK — The Invisible Deny That Kills Everything

Every ACL ends with an implicit "deny ip any any" — it's invisible, never shown in show access-lists, but it's always there. If you create an ACL with only a single permit statement and apply it to an interface, EVERYTHING else is denied. This is the most common ACL misconfiguration that causes outages.

DHCP Snooping — Stop Rogue DHCP Servers
ATTACK — Rogue DHCP Server (MITM Setup)

An attacker plugs a laptop running a DHCP server into any switch port. The rogue server broadcasts DHCP OFFERs faster than the real server. Clients get the rogue offer first and are assigned: attacker's laptop IP as their default gateway. All client traffic now routes through the attacker — complete MITM. The attacker forwards traffic to the real gateway (clients never notice).

MITIGATION — DHCP Snooping Trusted/Untrusted Ports

DHCP Snooping marks switch ports as TRUSTED (uplinks to real DHCP server) or UNTRUSTED (access ports facing users). DHCP OFFERs and ACKs from untrusted ports are DROPPED immediately. Clients can only receive offers from the legitimate server on trusted uplinks. Also builds the DHCP Snooping Binding Table (required for DAI).

Dynamic ARP Inspection (DAI) — Stop ARP Poisoning
ATTACK — ARP Poisoning / Gratuitous ARP

ARP has zero authentication. An attacker broadcasts a gratuitous ARP: "192.168.1.1 (gateway) is at AA:BB:CC:DD:EE:FF (attacker's MAC)". All hosts update their ARP cache. Now traffic destined for the gateway goes to the attacker. The attacker forwards it (MITM) while capturing everything. Also used for DoS: "gateway is at FF:FF:FF:FF:FF:FF" → traffic flooded.

MITIGATION — DAI Validates ARP Against Binding Table

DAI intercepts ARP packets on UNTRUSTED ports and checks: does this ARP's IP/MAC pair match the DHCP Snooping Binding Table? If no match → packet DROPPED and logged. Trusted ports (uplinks) are not checked. For devices with static IPs (not in binding table): create an ARP ACL.

Wireless Security Protocols

Concept — Wireless Security Evolution

Wireless is inherently more vulnerable than wired — signals travel through the air and can be captured by anyone in range. The encryption standard has evolved from completely broken (WEP) to highly secure (WPA3).

WEP — COMPLETELY BROKEN (NEVER USE)

ATTACK — WEP Cracked in Minutes

WEP uses RC4 cipher with a 24-bit IV (Initialization Vector) transmitted in cleartext. Only 16 million unique IVs exist — on a busy network, IVs repeat within hours. The FMS attack (2001) recovers the key by analyzing IV patterns. Aircrack-ng can crack WEP in under 5 minutes with enough packets. WEP is mathematically broken — no configuration makes it safe.

PMK (Pairwise Master Key) = derived from PSK (pre-shared passphrase). PTK = session key for unicast. GTK = group key for broadcast/multicast.

ATTACKS ON WPA2

KRACK (2017): Key Reinstallation Attack — forces nonce reuse in 4-way handshake → replay attacks. Now PATCHED in all major OSes.
PMKID Attack (2018): Capture PMKID from AP beacon (no 4-way handshake needed) → offline dictionary attack using hashcat. Mitigation: long random passphrase (20+ chars).
Evil Twin: Rogue AP with same SSID → deauth packets force clients to connect to attacker AP. WPA3 MFP prevents deauth attacks.

WPA3 — KEY IMPROVEMENTS

  • SAE (Simultaneous Authentication of Equals): Replaces PSK. Dragonfly key exchange (ECDH-based). Each guess requires online interaction with AP → rate-limited → offline dictionary attacks IMPOSSIBLE
  • Forward secrecy: Each session uses unique keys. Recording past traffic = cannot decrypt even if passphrase later compromised
  • Mandatory MFP: All management frames encrypted → deauthentication attacks blocked
  • OWE (Opportunistic Wireless Encryption): Open networks (cafes) get per-session encryption with no password — prevents passive eavesdropping
